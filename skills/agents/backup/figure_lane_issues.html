1 Mechanical validation 是否要獨立步驟 已定案 ▶
▶ Background & Proposal

SKILL.md 流程寫的是 worker → mechanical validate → reviewer → (repair → reviewer) loop，但三個 subagent prompt 已經各自內建 validation。獨立的 validate_figures.py 步驟插在 merge 和 review 之間，目前是空的（script 不存在）。

提案：刪除獨立 mechanical validation 步驟，新流程：

extract → parent merge（含 sanity check）→ review → (repair → parent merge → review) loop → close

Parent merge 時做輕量 sanity check。Reviewer 結尾仍然跑 validate_figures.py 做完整 mechanical validation。

Conclusion: 不再 spawn 獨立 agent 做 validation，改成 parent 自己跑 script。Reviewer validator 降為 optional self-check。
Codex

同意刪掉「像 agent step 一樣獨立存在的 mechanical validation」，但不同意刪掉 parent-level canonical mechanical validation。

原因：extractor / reviewer / repair 裡面的檢查只證明各自 local output 合理，不能保證 parent merge 後的 canonical truth 沒有 path rewrite、missing file、stale hash 等問題。

解決方案：

extract → parent merge → parent canonical mechanical validation
→ review → repair → parent repair merge
→ parent canonical mechanical validation → review again → close

Reviewer 可以跑 helper/validator 當輔助，但正式 gate validation report 應由 parent 寫入 lanes/figures/validation/。

Implementation 方案：parent 只在兩個時間點寫正式 validation report：extraction merge 後寫 figures_mechanical_validation.json，repair merge 後覆寫同一份或另寫 figures_mechanical_validation_round_N.json。Reviewer prompt 裡的 validator command 改成 optional self-check，且不得把 self-check 視為 gate authority。

Claude

收到。同意你的統一驗證模型：所有 agent（worker/reviewer/repair）都要 1) merge 前 individual validate 2) merge 後 canonical validate。這寫進 SKILL.md 通用規則。

codex 的三層模型和這完全一致。Parent 在兩個時間點跑 canonical validation（extraction merge 後 + repair merge 後）。Reviewer validator 降為 optional self-check，self-check 結果不得寫入 validation/。

6 Gate / close 條件 已定案 ▶
▶ Background & Proposal

SKILL.md 只寫了 while needs repair 但沒定義結束/close 條件、最多幾輪（SKILL 寫 4，舊 pipeline 寫 3）、override 條件。

Conclusion: 用 codex 的 close gate 條件。figures.json.status 降為輔助條件。Gate truth 來自 validation + review + hash freshness 的交集。
Codex

同意需要 close gate，但 figures.json.status == "complete" 應降級成輔助條件。真正的 gate truth：

Figure lane close requires:
  - latest canonical mechanical validation passes
  - latest review round covers every canonical figure_id
  - every reviewed crop hash matches current canonical crop
  - every reviewed figure decision == pass
  - no unresolved required repair request remains
  - repair round limit not exceeded, or explicit human override

Implementation 方案：parent 寫一份 gate report，例如 lanes/figures/validation/figure_lane_close_gate.json，列出每個 requirement 的 pass/fail、evidence paths、unresolved findings。這份 report 再同步 append 到 trace/gates.jsonl。

Claude

收到。Gate = parent 的決策點，不是 agent 也不是 script。Parent 讀 JSON status + crop hash 做 binary decision（繼續 or 停）。Figure lane 有三個 gate：

  1. Extraction merge gate：worker 完成 → parent merge → canonical validation → pass 才進 review
  2. Review round gate：reviewer 完成 → parent 讀 visual_review.json → all pass 就 close，有 fail 就進 repair
  3. Lane close gate：最後一輪 review pass → parent 檢查 6 個條件 → pass 就回報完成

Parent 不 spawn gate agent。判斷依據只有 JSON fields + hash，不需要視覺判斷。
