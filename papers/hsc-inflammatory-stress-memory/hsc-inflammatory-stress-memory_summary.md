# 主線
作者想知道人類造血幹細胞在反覆發炎之後，有沒有像免疫細胞一樣保留『記得發炎這件事』的痕跡。他們把人類臍帶血幹細胞種進小鼠體內，反覆給它們打發炎訊號，再讓它們休息十週，然後用單細胞多體學去看它有沒有變得不一樣。結果發現，幹細胞分裂出兩群，其中一群把上次發炎的記憶寫進染色質與基因表現，從此變得更不愛工作；這群帶記憶的幹細胞會把發炎傾向傳給孩子細胞，並且在嚴重 COVID-19 後、老人、鐮刀型紅血球病、以及與白血病相關的『複製性造血』突變者身上都更多，還和較高的死亡風險相關。

# 摘要
人體血液裡每天有三百萬個新細胞，全部來自骨髓深處少數的造血幹細胞 (HSC)。作者好奇一件事：這些幹細胞被一輩子大大小小的感染影響後，會不會像免疫細胞一樣『記得發炎』，並把這份記憶寫進自己的長期狀態？但活人骨髓沒辦法反覆抽樣追蹤十幾週，這件事在人身上看不出來。為了把這件事拆開來看，他們把人類臍帶血裡的幹細胞挑出來，種進沒有免疫系統的小鼠骨髓裡，讓人類幹細胞在小鼠體內生長。接著對小鼠連打兩輪『發炎訊號』：一種是身體在感染或老化時會自己釋放的警報分子 (TNF)，另一種是模擬細菌感染的細菌毒素 (LPS)；兩輪中間隔幾週，最後讓小鼠休息十週，模擬『反覆感染後逐漸恢復』的場景。

在這個模型裡，作者用一種叫單細胞多體學 (scMultiome) 的技術，一次量了將近三萬個幹細胞，同時記錄哪些基因在運作、以及哪些段落的 DNA 被『鬆開』可以使用——後者反映這個細胞長期保留了哪些工作守則。原本以為是一團的幹細胞，居然清楚分成兩塊區域，作者命名為 HSC-I 與 HSC-II。HSC-II 這群長期維持兩個特徵：一是更深的休眠（很少分裂、產出很少新血球），二是把兩位專門啟動發炎反應的指揮官蛋白 (NF-κB 與 AP-1) 能接觸的 DNA 預先打開，等於把指揮台架好待命。最關鍵的是，這兩個特徵在打過 TNF 或 LPS 的小鼠裡比沒打過的更明顯。也就是說，它們不是天生差異，而是上一次發炎刻下來的長期痕跡。作者於是把這群帶著痕跡的細胞改名為 HSC-iM (HSC inflammatory memory，帶發炎記憶的幹細胞群)。

光在小鼠模型裡看到還不能算數。作者把 HSC-iM 的分子特徵清單拿去比對真實病人：嚴重 COVID-19 出院兩到四個月後的患者、有慢性發炎的鐮刀型紅血球病孩童、以及不同年齡的正常成人骨髓。結果在這些情境下，符合 HSC-iM 特徵的幹細胞都明顯增加；越老或慢性發炎越重，這群細胞累積得就越多。換句話說，HSC-iM 不是小鼠實驗的人造產物，而是真實對應一輩子被反覆感染、發炎『燒過』的人類骨髓。

接下來作者把 HSC-iM 連到一件臨床上很重要的事：複製性造血 (CH)。中老年人骨髓裡常會出現幾顆帶白血病前期突變的幹細胞，最常見的是 DNMT3A 或 TET2 兩個基因壞掉；這些細胞會慢慢佔比變高，跟未來罹患白血病與心血管疾病、甚至整體死亡率都有關。在這些 CH 患者骨髓裡，作者看到兩件耐人尋味的事。第一，連沒帶突變的幹細胞也都同時往 HSC-iM 狀態靠攏——也就是發炎開關長期半打開、休眠特徵變深——代表這些人本身就是發炎記憶累積較多的人。第二，DNMT3A 或 TET2 突變幾乎只在 HSC-iM 這群裡引發顯著的基因改變；而且突變的效果，是把 HSC-iM 原本『懶得分裂』的鎖打開，逼它從後台跑出來大量生產髓系細胞。這條因果線可能正是 CH 克隆越長越大、之後又增加白血病風險的機制之一。

HSC-iM 自己安靜地躲在骨髓深處還不算嚴重，問題是它生出來的子代細胞會不會也帶著這份發炎脾氣。為了回答這題，作者用了一個巧妙設計：每位捐贈者的 DNA 上都散落一些只屬於自己的小拼字差異，子代細胞會原樣繼承，等於是自帶的『天然條碼』。靠這個條碼，他們把實驗裡 8 萬多個下游細胞依血緣分組，做了一場細胞家譜溯源。結果發現：源自 HSC-iM 的家族，從祖細胞、分化中的前驅細胞、到成熟的單核球，全程都帶著比較強的發炎特徵；源自 HSC-I 的家族則沒有。也就是說，這份『記得發炎』的脾氣會原封不動地一路傳下去。

最後一塊拼圖是要看這件事在活人身上有沒有意義。作者在四百多位志願者的外周血裡，套用一個臨床上常用的『綜合死亡風險指標』(IRS)，這個分數靠常規血球與生化檢查算出來，分數越高表示未來死亡風險越高。在年輕族群裡，淋巴球與單核細胞身上 HSC-iM 特徵越強，IRS 也越高。這暗示『記得發炎』這件事可能不只是顯微鏡底下的分子痕跡，也會反映在長期健康上。

在被這個故事說服之前，幾個邊界仍要說清楚：小鼠模型只追蹤約二十週，看不到真正『一輩子』累積的效果；HSC-iM 與 COVID-19、老化、鐮刀型紅血球病、CH、死亡風險之間目前都是相關性，還不是直接因果；而且作者目前還沒有辦法只靠一個表面標記就把活的 HSC-iM 挑出來，這也限制了未來更直接的功能驗證。
