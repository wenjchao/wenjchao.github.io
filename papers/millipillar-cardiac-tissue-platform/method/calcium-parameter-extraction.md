# 主線
將 GCaMP6f / Fluo-4 鈣影片轉換成可跨研究比較的量化指標（Tau、FWHM、FW90M、Contract 50/90、Relax 50/90、SDRR），並提供統一程式碼以消弭「不同實驗室取樣定義不同」的歧異。

# 技術解析
鈣影片每張影格本來是一張二維亮度地圖。程式先把每張影格內所有像素亮度求平均，把這張影格壓成一個數字 (空間平均 spatial averaging)，整段影片就變成一條亮度隨時間變化的曲線——所有後續指標都從這條曲線量。但有個陷阱必須先處理：GCaMP6f / Fluo-4 在持續藍光激發下，螢光分子會逐張被打壞、整段基線像潮水退去般慢慢下滑，這叫光漂白 (photobleaching)。如果不校正，後段峰值看似變矮、衰退尾巴看似變長，會把組織誤判為「鈣回收變差」、得到跟成熟化完全相反的結論；校正方法是先抓整段緩慢下滑的趨勢線、再把每張影格亮度除以這條線拉平，留下純粹的鈣訊號漲落。校正後的曲線上有一連串鈣脈衝，程式把每個當作一座尖塔來量六個指標，方便起見可分四群來看：上升速度看 Contract 50/90 (亮度從靜止上升到峰值 50% 或 90% 的時間)，反映鈣湧入有多快；高度與胖瘦看 FWHM (半高處橫切的塔寬) 與 FW90M (九成高處橫切的塔頂寬)；下降速度看 Relax 50/90 (從峰值退回 50% 或 90% 的時間) 與 Tau (退到原本 1/e ≈ 37% 的時間)——Tau 不是隨便挑的閥值，因為鈣回收近似一階指數衰退 A·exp(-t/τ)，當 t = τ 時剛好降到 1/e，所以 Tau 直接對應細胞內鈣抽運蛋白 (例如 SERCA) 的真實工作速率；節律規不規律看 SDRR (相鄰兩拍峰頂間距標準差)，類似心律不整指標。一次算這麼多並非貪心：心肌成熟化牽涉好幾個獨立過程，上升快、回收快、拍寬縮短、節律穩各自反映不同機制；只挑一兩個會錯失「上升正常但回收變慢」這種細微差異。為什麼要選 CalTrack 風格？過去不同實驗室對「Contract 時間」可能算 0–80%、10–90% 或 0–90%，跨論文沒辦法直接比；作者把 Psaras et al. Circ. Res. 2021 的 CalTrack 與 Sala et al. Circ. Res. 2018 的 MUSCLEMOTION 定義的鈣瞬變指標一次算齊、用同一份 Python 程式輸出表格，等於把參數定義寫進開源 code 裡，下游 reviewer 可逐欄對應、不必再爭論取樣定義。彙整與同步偵測也都自動化：1 Hz 鈣錄影 15 秒可抓約 15 個脈衝，把它們的指標平均成單一代表值，比只看一拍可靠；ET/MCR 部分則由程式自動掃描電壓或頻率，每換一個參數就生成一段 trace、檢查鈣峰是否與電脈衝一一對應，失同步處即 ET/MCR 臨界點。最後一個必須記住的陷阱是螺旋波 (spiral wave)：組織裡有一小塊區域自己高頻自激、整體 trace 看起來仍規律，但其實是局部失控；程式只看空間平均後的時間序列，分不出哪邊在帶頭，會誤判為「跟得上刺激」。所以作者強烈建議分析完還要回去看原始影片人眼確認，特別在 ET/MCR 邊界附近，避免拿一條假成熟化的曲線去做統計。

# 工具/方法/材料
- **Spatial averaging**：把每張影格內所有像素亮度求平均壓成一個數字，整段影片轉成一維時間序列。
- **Photobleaching correction**：抓出整段亮度基線下滑趨勢線並除掉它，消除藍光長時間激發造成的螢光分子破壞偽影。
- **Tau (鬆弛常數)**：鈣濃度從峰值退到 1/e ≈ 37% 的時間，對應一階指數衰退時間常數，直接反映 SERCA 等鈣抽運速率。
- **FWHM**：Full width at half-maximum，在峰值一半高度橫切的塔寬，量整體鈣瞬變持續時間。
- **FW90M**：Full width at 90% maximum，在峰值九成高度橫切的塔頂寬，對峰頂胖瘦更敏感。
- **Contract 50/90**：亮度從靜止上升到峰值 50% 或 90% 所需時間，量鈣湧入速度。
- **Relax 50/90**：從峰值衰退回峰值 50% 或 90% 所需時間，量鈣回收速度。
- **SDRR**：Standard deviation of RR intervals，相鄰兩拍峰頂間距的標準差，量節律規不規律。
- **CalTrack (Psaras 2021)**：心肌領域近年廣為引用的開源鈣訊號分析框架，明確規範指標數學定義；作者以 Python 重寫採同一規格。
- **MUSCLEMOTION (Sala 2018)**：另一套常被引用的鈣/收縮影像分析工具，作者一併沿用其鈣瞬變定義。
- **ET/MCR 同步偵測**：程式逐頻率掃描，比對電脈衝與鈣峰是否一一對應；失同步處即 excitation threshold 或 maximum capture rate。
- **Spiral wave 人工複核**：螺旋波局部高頻自激不會破壞 trace 規律性、自動程式分不出來；作者建議錄影完仍要人眼回看原始影片。

# 與此篇文章的關係
在《milliPillar: A Platform for the Generation and Real-Time Assessment of Human Engineered Cardiac Tissues》這篇文章中，作者為了讓 GCaMP6f / Fluo-4 鈣影片在跨實驗室層級能被直接比較，採用「空間平均 + 光漂白校正 + CalTrack/MUSCLEMOTION 風格指標」的 Python 統一程式碼。它解決了過去 Contract 時間定義各家自訂、跨論文無法逐欄對應的瓶頸，把鈣影片轉成 Tau / FWHM / Contract / Relax / SDRR 表格，餵給下游電刺激成熟化前後的 paired t-test 統計分析。
