# 核 (the core) 作為 value capture 的解概念，並用 added-value bounds 取得 MB / CV

1. 引用自哪篇 paper: value-capture-with-frictions
2. Outline (任務主線): 核 (the core) 作為 value capture 的解概念，並用 added-value bounds 取得 MB / CV
3. Method:
   這個模塊把『誰能分到多少錢』的可行區間用解概念精確界定。核 (the core) 是合作賽局的標準解概念：給定 § 2A 的特徵函數 v(·)，核蒐集所有同時滿足兩條件的分法 (x_B, x_1, x_2)——效率 ∑_{i∈N} x_i = v(N)（餅剛好分完）；穩定 ∑_{i∈G} x_i ≥ v(G) 對所有真子集 G ⊂ N 成立（任何子集裡的人加起來不能拿得比自己幹還少，否則他們會集體脫離，稱為 no blocking coalition）（paper Eq. 7）。套到三玩家情境，核的解集合是（paper Eq. 9）：強供應商 x_1 ∈ [0, V_1 − V_2]、弱供應商 x_2 = 0、買方 x_B ∈ [V_2, V_1]——弱供應商沒有加值故得 0，買方有 V_2 作為外部選項鎖定下限。MB (minimum bound, 最小可接受) 是核中該玩家能拿到的最低值；CV (ceiling value, 最大可接受) 是核中能拿到的最高值，剛好等於該玩家的加值 v(N) − v(N \ {i})——強供應商 MB = 0、CV = V_1 − V_2；買方 MB = V_2、CV = V_1。在雙玩家情境（單供應商+買方，bilateral monopoly），核退化為 x_i ∈ [0, V_i]、x_B ∈ [0, V_i]——任何加起來等於 V_i 的分法都在核裡，分餅完全交給後續 α-加權處理。
   MB 與 CV 不是憑空挑出來的數字，而是穩定條件的直接代數推論。為什麼買方的 MB = V_2？穩定條件套用在子集 G = {B, 2} 時要求 x_B + x_2 ≥ V_2，因為 x_2 = 0，剩下的 V_2 全部落在買方頭上——直觀地說，買方隨時可對強供應商說『不爽我就跟對手做生意，我至少能拿 V_2』，這就是 V_2 作為買方外部選項 (outside option) 鎖定下限的數理機制。為什麼 CV = 加值？把穩定條件套用在 G = N \ {i}（除了自己以外所有人），要求 ∑_{j ≠ i} x_j ≥ v(N \ {i})；配上效率條件 ∑ x_j + x_i = v(N)，立刻得到 x_i ≤ v(N) − v(N \ {i})，正好是 i 的加值。物理意義是：每個玩家最多只能爭取『沒有他大家少做出多少錢』這個額外貢獻，超過就會逼其他人離開大聯盟，自己反而沒得分。CV = 加值是穩定條件強加的天花板。
   為什麼用核作為 value capture 的解概念，而不挑 Shapley value 或 nucleolus？因為本論文策略意涵的核心是『α-加權』——買方與供應商之間的相對議價力可連續調整 α ∈ [0,1]。核給出的區間 [MB, CV] 正好提供這個 α-加權的可調空間：α = 0 時退化為 Bertrand（買方拿到 CV、供應商拿到 MB）、α = 1 時相反；如果一開始就鎖死成單一點，後續就無法做議價力的比較靜態與 Proposition 6 的 ∂Π / ∂α 分析。這個設計也順手化解 Aumann (1985) 與 Lippman & Rumelt (2003) 對核『不寫實』的批評：他們指出核解出來太極端（劣勢供應商完全擠到 0）。作者的回應是核的區間其實已內含『Bertrand 之上再讓買方繼續議價』的彈性——α = 0 退化為純 Bertrand 結果（買方拿天花板 V_1），α > 0 則買方少拿一些、強供應商分到 (1−α)(V_1 − V_2)。所以核並不是『不寫實』，而是『把議價力先壓到 0 的特例』。
   兩個地方一失手結論會崩。第一，漏掉某個子集 G 的穩定不等式：穩定條件要對『所有』真子集 G ⊂ N 都成立，漏掉一個（例如忘了 G = {B, 2}）就可能宣告『核包含一個讓 G 想集體脫離的分配』，那其實已經不是核——它違反 no blocking coalition。對三位玩家來說子集只有 2^3 − 1 = 7 個，全部得列出來檢查；玩家數多時這份檢查表會爆炸，這也是為什麼這套核分析方法只能用在玩家數少的乾淨模型上。第二，在雙玩家情境（bilateral monopoly）仍堅持『核獨自定出分配』：核退化成整段 [0, V_i] 區間，任何加起來等於 V_i 的分法都合法，模型會給出無窮多個合法預測，根本無法做比較靜態。所以雙玩家情境必須靠 α-加權再走一步把點選定——核給出『可行區間』而非『唯一分配』，下一步的 α-加權不是裝飾，而是讓模型可被驗證的必要步驟。
4. 工具與材料:
   - **核 (the core)**: 合作賽局解概念：滿足效率（餅剛好分完）與穩定（無子集想脫離）兩條件的所有分法集合。
   - **效率條件 ∑_{i∈N} x_i = v(N)**: 所有玩家分到的錢加總等於大聯盟總價值。
   - **穩定條件 ∑_{i∈G} x_i ≥ v(G)**: 對任何子集 G 都成立的不等式，避免子集集體脫離大聯盟（no blocking coalition）。
   - **加值 (added value)**: v(N) − v(N \ {i})，玩家 i 對大聯盟的邊際貢獻，等於其 value capture 的天花板 CV。
   - **MB (minimum bound)**: 該玩家在核中能拿到的最低值，即 walk-away 底線（買方的 MB = V_2 由外部選項鎖定）。
   - **CV (ceiling value / maximum value capture)**: 該玩家在核中能拿到的最高值，等於其 added value（強供應商 CV = V_1 − V_2，買方 CV = V_1）。
   - **雙玩家核 (bilateral monopoly)**: 單供應商+買方時核退化為整段 [0, V_i]，無法唯一定分配，須交給 α-加權。
   - **α = 0 退化為 Bertrand**: 把核重新詮釋為『Bertrand 之上再讓買方議價』的關鍵極限，化解 Aumann / Lippman-Rumelt 對核的批評。
5. 與此篇文章的關係:
   在《Value Creation and Value Capture with Frictions》這篇文章中，作者為了精確界定每位玩家可分到的 value capture 區間，採用了合作賽局的核 (the core) 並用穩定條件直接讀出 MB 與 CV 上下界。這個方法解決了既有 value-based 文獻常把核當『單一極端結果』而被批評不寫實的瓶頸，並把 § 2A 特徵函數產出的 v(·) 表轉成下游 α-加權與 Proposition 1–7 比較靜態所需的閉式 [MB, CV] 區間。
