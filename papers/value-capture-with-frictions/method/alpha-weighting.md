# α-參數化的 value capture 點預期 (Mapping the core to a point via bargaining-power weights)

1. 引用自哪篇 paper: value-capture-with-frictions
2. Outline (任務主線): α-參數化的 value capture 點預期 (Mapping the core to a point via bargaining-power weights)
3. Method:
   在「一買方、兩供應商」情境下，核 (the core) 把總價值 V_1 切成三份，但只把每份鎖在一段區間裡：強供應商分到的數可以是 0 到 V_1 − V_2 之間任何一個值；弱供應商必拿 0；買方拿到的數可以是 V_2 到 V_1 之間任何一個值（paper Eq. 9）。在「一買方、一供應商」雙寡頭情境下，核連區間都鎖不住，只說兩人加起來等於 V_i（paper Eq. 8）。會這麼鬆散，是因為核只設了「下限」（任何小團體拆出去都不能更賺）和「總和」（加起來等於 V_1），剩下的價值怎麼分模型本身沒答案，直覺上就像談判桌的合法成交價可以是一段區間，怎麼落點要看雙方嘴皮子有多硬。為了把這片區間收成一個點，作者對每個玩家在自己的核區間 [下界, 上界] 上挑一個固定位置：期望 value capture = 下界 + α_i × (上界 − 下界)。α_i = 0 代表這個玩家永遠拿到區間底、α_i = 1 代表永遠拿到區間頂。本文把買方權重設 α_B = α、兩個供應商都設成 α_1 = α_2 = 1 − α，讓 α 可以直接讀成「買方相對於賣方有多強勢」。代進三玩家核後得到 Π_1 = (1 − α)(V_1 − V_2)、Π_2 = 0、Π_B = V_2 + α(V_1 − V_2)（paper Eq. 11）；雙玩家情境化成 Π_i = (1 − α)V_i、Π_B = α V_i（paper Eq. 10）。從「一片區間」變成「一個數」，後面就能直接對 α、V_i、f 取偏微分。
   α 之所以是這個用法，是因為它的議價力解釋有理論背書。Brandenburger & Stuart (2007) 一開始引入 α 只把它定義成「玩家期望落在核區間哪個位置」，並沒承諾這個位置一定對應到真實談判力。Chatain & Zemsky (2007, Assumptions 1–3) 進一步證明：只要模型滿足三個條件——沒有產能限制、玩家之間沒有外部性、added value 之間沒有互補性——α 的位置就會剛好等於真實討價還價分到的比例。本文模型三條件全中（每個供應商可以服務任意多買方、每對買賣方的交易不影響其他配對、強弱供應商的 added value 不互相加成），所以 α 的「相對議價力」標籤是有理論背書的（footnote 10）。換成 Shapley value 或 Nash bargaining 為什麼不行？兩者都把議價力鎖死成一個固定數，不能當作連續可調的旋鈕；α 是從 0 到 1 連續變動的單參數，這讓作者後面能對 α 做偏微分（例如 Proposition 6 的 ∂f_L^D/∂α）。同時，把兩供應商權重設成同一個 1 − α 是宣告 α 是 industry-level 的旋鈕、不是 firm-level 的旋鈕，這樣才能讓「α 一動，所有人 Π_i 同步移」的整體比較靜態成立。
   α 把核收成點還有一個重要的副產品：把 α = 0 代進三玩家公式，會得到強供應商拿 V_1 − V_2、買方拿 V_2，這正好就是經典 Bertrand 價格競爭的結果——弱供應商被殺到零、強供應商只賺得到「比對手多做的那一小塊」、買方把弱對手的價值整碗端走。雙玩家情境 α = 0 退化成 Π_i = V_i、Π_B = 0。換句話說，核 + α 並不是一個跟傳統割裂的奇怪框架，它在 α = 0 的極限直接接回經濟學裡最熟的競爭基準。把 α 從 0 慢慢開高，等於「在 Bertrand 之上再讓買方有空間繼續跟強供應商談」，這正是本文回應 Aumann (1985) 與 Lippman & Rumelt (2003) 對核「太極端」批評的關鍵錨點。
   如果跳過 α 加權、直接拿核區間做後續分析會壞在哪？偏微分需要在「同一個落點」上才能定義（你必須先選定「強供應商落在區間哪裡」這個值才能談「V_1 增加一點，這個值如何動」），否則符號根本算不出。沒有 α 收緊區間，Proposition 1（own / cross friction 偏微分）、Proposition 2 的倒 U 型、Proposition 3 的 Makadok cross-partial 全都會失去落點，整篇 comparative statics 章節直接報廢。另一個更隱蔽的失敗模式發生在 α 仍存在但 Chatain & Zemsky (2007) 三條件被破壞時：α 仍能把區間收成點，但這個點不再剛好等於玩家真實談判分到的份額。命題裡寫「買方議價力上升」就變成空話——你動的只是「期望落在區間哪裡」這個技術參數，跟真實議價力脫鉤。
4. 工具與材料:
   - **核 (the core)**: 合作賽局的解概念，給出「沒有任何小團體能拆出去更賺」的所有合法分配；本文中通常是一段區間而非單點。
   - **α-weighting (α ∈ [0,1])**: 在每個玩家的核區間 [下界, 上界] 上挑一個固定位置：期望 value capture = 下界 + α_i × (上界 − 下界)。
   - **α_B = α、α_1 = α_2 = 1 − α**: 本文設定買方權重為 α、兩供應商共享權重 1 − α，使 α 解釋為 industry-level 的買方相對議價力。
   - **三玩家期望 value capture (paper Eq. 11)**: Π_1 = (1 − α)(V_1 − V_2)；Π_2 = 0；Π_B = V_2 + α(V_1 − V_2)。
   - **雙玩家期望 value capture (paper Eq. 10)**: Π_i = (1 − α)V_i；Π_B = α V_i，對應 bilateral monopoly 情境。
   - **Bertrand 極限 (α = 0)**: α = 0 時公式退化成 Bertrand 價格競爭結果，作為核 + α 框架接回傳統經濟學的錨點。
   - **Chatain & Zemsky (2007) Assumptions 1–3**: α 可被合法解釋為相對議價力的三個充分條件：無產能限制、無外部性、added value 無互補性。
5. 與此篇文章的關係:
   在《Value Creation and Value Capture with Frictions》這篇文章中，作者為了把後續所有比較靜態（Proposition 1–6）建在一個可以做偏微分的閉式期望函數上，採用了 Brandenburger & Stuart (2007) 的 α-weighting：把核給出的「一段合法分配區間」透過買方議價力 α 收斂成一個明確的點。這個方法解決了「核太鬆散、區間無法直接微分」的瓶頸，吃入核區間下界與上界、產出 Π_1, Π_2, Π_B 的閉式期望，直接餵給下一節的 f-comparative statics。
