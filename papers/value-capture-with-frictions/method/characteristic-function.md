# 基準特徵函數 (Characteristic function for the base buyer–supplier game)

1. 引用自哪篇 paper: value-capture-with-frictions
2. Outline (任務主線): 基準特徵函數 (Characteristic function for the base buyer–supplier game)
3. Method:
   作者把市場壓縮成最小單位：一個買方 B 和兩個供應商 1、2，然後寫一張「每一種人湊在一起能做出多少生意」的價目表——這張表就叫特徵函數 (characteristic function)。表的內容很直白：任何人自己一個獨來獨往沒生意 (v(B)=v(1)=v(2)=0)；買方配上供應商 1 能做 V1 元；配上供應商 2 能做 V2 元；三個人都湊在一起也只能跑效率比較高的那條線，所以仍是 V1 元。兩家供應商賣的是同一件事但一家做得比較好 (vertically differentiated)，這種「同類但有高下之分」的設定讓「優勢供應商多創造的價值」直接等於 V1 − V2，是個非負的乾淨數字。任何人自己單獨設為 0 是在明說：這張表上每一塊錢都是「結盟」才生出來的，後續『誰該分多少』的問題從根本上鎖在結盟貢獻上。
   特徵函數的威力在於：它把『價值怎麼產生』這個問題完整外包到一張表上，後續無論用哪一種「該怎麼分」的解法（核、Shapley value 等）都只要把這張表當輸入就能算。所以作者只要動表本身——後面要加入摩擦，就讓表的內容隨機切換；要加入進入決策，就在表上補一筆固定成本——解法本身不必動。這就是為什麼這個基準骨幹寫對了，後面五個擴充都能順著疊。
   為什麼挑「兩家垂直差異」這個最簡設定？因為它是讓核能用紙筆手算的最小規格——三家以上供應商會把可能的子聯盟組合炸開，後面邊界寫不出閉式；換成橫向差異（一家賣紅、一家賣綠，買方各有偏好），『誰比較強』會隨買方口味來回切換，後續所有比較靜態都要多開一個維度。同樣關鍵的是，這張表的『設計責任』比想像中重：如果特徵函數直接允許所有買賣方自由湊團，核會把劣勢供應商當成「永遠可以被叫來當備胎」——買方只要拿『另一家也能做』當談判牌，就能把優勢供應商的利潤壓掉一大塊，比 Bertrand 殺價還殘酷 (Aumann 1985)。先前唯一的緩解辦法是加入產能限制 (capacity constraint)，但那會把分析鎖死在 Ricardian 租金邏輯。作者的修法不是改解概念，而是把『可結盟的人有時根本連不上線』寫進特徵函數本身——這就是下一個子項要做的事。
4. 工具與材料:
   - **characteristic function (特徵函數)**: 把『每一種玩家組合能做出多少生意』一一列出的價目表，是合作賽局的標準輸入。
   - **vertically differentiated**: 兩家供應商賣的是同一件事但一家做得比較好，效率排序固定為 V1 ≥ V2。
   - **V1, V2**: 高效率供應商與低效率供應商分別與買方配對所能做出的總價值。
   - **added value**: 一個玩家加入聯盟之後總價值多了多少；優勢供應商的 added value 直接等於 V1 − V2。
   - **Bertrand 殺價**: 兩家賣家被買方拉去同台殺價、把價格壓到成本的標準結果；本文核在無摩擦時甚至比這還殘酷。
   - **capacity constraint**: 舊文獻緩解核過度競爭的傳統手段——限制供應商產能，但代價是把分析鎖死在 Ricardian 租金。
5. 與此篇文章的關係:
   在《Value Creation and Value Capture with Frictions》這篇文章中，作者為了把產業競爭強度與廠商異質性整合進同一套合作賽局，先用『基準特徵函數』寫出最精簡的兩供應商一買方價值地圖。它解決的瓶頸是：先前的 value-based 模型沒寫清楚『誰結盟能做多少生意』，導致後續核的計算被過度競爭吃乾抹淨。這張價目表是後續摩擦、進入、投資三個擴充的共用地基，產出一組 v(B,1)=V1、v(B,2)=V2、v(B,1,2)=V1 的封閉公式直接交給下一個子項『摩擦機率化』疊上隨機性。
