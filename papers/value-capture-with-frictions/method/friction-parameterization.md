# 摩擦的機率化 (Friction parameterization via urn-ball random matching)

1. 引用自哪篇 paper: value-capture-with-frictions
2. Outline (任務主線): 摩擦的機率化 (Friction parameterization via urn-ball random matching)
3. Method:
   作者引入一個摩擦參數 $f \in [0, 1]$，代表『買方根本沒注意到某家供應商存在』的機率。技術上等於在賽局裡多塞一個隱形玩家 (nature)：他在交易開始前先擲骰子決定『今天買方看得到哪幾家供應商』，然後才進入後續的核計算。每家供應商被看見與否互相獨立，這種結構承襲自勞動經濟學的『球丟進罐子』隨機配對模型 (urn-ball matching, Petrongolo & Pissarides 2001)。獨立事件相乘就得到四種情境：兩家都被看見的『雙寡頭』機率 $(1-f_1)(1-f_2)$、只有強者被看見的『1 號獨佔』機率 $(1-f_1)f_2$、只有弱者被看見的『2 號獨佔』機率 $f_1(1-f_2)$、兩家都沒被看見的『買方無人可買』機率 $f_1 f_2$。第四情境的價值對應『不買、垂直整合、改用 generic 替代品』三選一，留給作者解釋彈性。
   為什麼這一個 f 就能同時攜帶三股力量？把 f 拉一格就同時動了三件事：雙寡頭機率 (1−f)^2 變小（rivalry 強度下降）、單一家獨佔機率 (1−f)·f 變大（市場效率受擾）、買方完全沒生意機率 f^2 變大（外部選項被觸發）。一個旋鈕同時轉三個音量鍵，這就是後續每個 Proposition 的比較靜態都能化成對 f 偏微分的根本理由。另一個關鍵設計：f 改寫的是『今天有效的特徵函數版本』而不去改 v 的數值大小——作者描述的不是『生意做出來比較少錢』而是『有些人根本見不到面』，這個區分讓後續解釋『摩擦如何鬆綁劣勢供應商』時很乾淨：劣勢供應商有生意不是因為強者變弱，而是因為強者今天根本不在場。
   為什麼後續分析多用 $f_1 = f_2 = f$？異質摩擦保留給最開頭的 Proposition 1，讓讀者看到『某家自己摩擦變高傷自己、對手摩擦變高反而幫到自己』這種 cross-firm 互動結構。一旦進入主秀，作者就把 f1=f2 鎖在一起，這時 f 直接代表『整個行業的摩擦強度』，對應 Porter 五力中的 rivalry——分析維度從二維壓到一維，後續 entry / resource 章節才可能維持閉式。回頭看為什麼非得把 rivalry 寫成機率化的連續參數：若只說『假設競爭很激烈』，每換一個情境就要重寫一次模型，沒辦法在同一條座標軸上讀出『摩擦從 0 拉到 1 時，行業利潤、進入結構、資源投資怎麼變』。寫成 f 之後，作者才能對每個結果做偏微分，也才能把 industry / firm / resource 三層分析鎖在同一條軸上。
4. 工具與材料:
   - **摩擦參數 f**: $f \in [0,1]$，代表買方根本沒注意到某家供應商存在的機率；f 越大代表彼此越容易連不上線。
   - **urn-ball matching**: 勞動經濟學處理『一群獨立尋找者各自隨機命中目標』的標準模型 (Petrongolo & Pissarides 2001)，本文借來描述買賣方獨立偶遇。
   - **nature (隱形玩家)**: 賽局內額外的『先擲骰決定今天看得到哪幾家供應商』的玩家，把摩擦寫成隨機事件。
   - **Table 1 四情境**: 雙寡頭 (1−f1)(1−f2)、強者獨佔 (1−f1)f2、弱者獨佔 f1(1−f2)、買方無人可買 f1·f2。
   - **同質摩擦 f1=f2=f**: 主秀章節的簡化假設；f 直接代表整個行業的摩擦強度，對應 Porter 五力中的 rivalry。
   - **Proposition 1**: 異質摩擦下的命題：own-friction 偏微分為負、cross-friction 偏微分為正、摩擦為策略替代品。
5. 與此篇文章的關係:
   在《Value Creation and Value Capture with Frictions》這篇文章中，作者為了把 Porter 五力裡『rivalry 強度』從定性概念升級為可比較靜態的連續變數，採用了 urn-ball 隨機配對來把摩擦機率化。這解決了先前 value-based 模型只能用『加產能限制』來緩解核過度競爭的瓶頸，把摩擦寫進特徵函數本身就能讓同一套模型在 industry、firm、resource 三層通用。這個方法吃 2-A 的基準特徵函數為輸入，產出一組四情境機率權重，直接交給後續核 + α-weighting 計算閉式 value capture。
