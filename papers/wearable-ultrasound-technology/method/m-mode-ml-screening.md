# M-mode 影像 + Machine Learning 自動篩選

1. 引用自哪篇 paper: wearable-ultrasound-technology
2. Outline (任務主線): M-mode 影像 + Machine Learning 自動篩選
3. Method: 
   傳統 B-mode 是 2D 灰階切面，而 M-mode 是把一條固定方向的掃描線隨時間排成一張紙——縱軸深度、橫軸時間，動脈每跳一下管壁就撐大再縮回，M-mode 上會出現上下起伏的條紋，這條紋寬度隨時間的變化就是動脈直徑波形 (arterial diameter)。手腕貼片這類監測最需要量的就是這條波形。問題是穿戴貼片貼定後不能像技師那樣微調探頭，使用者一活動，動脈在皮膚下會偏移幾毫米；如果只靠一條掃描線，動脈一偏就丟訊號。作者用 Lin et al. 2024 的 32-element 陣列同時各掃一條 M-mode，相當於把 32 條平行掃描線並排撒在手腕上——只要動脈還在貼片底下，其中至少一條就會剛好對準。剩下的問題變成：誰來從 32 條裡挑出對準的那一條？
   這就是 ML 上場的位置。作者先收集大量手腕 M-mode 影像，由人標好「這條對準動脈、這條沒對到」當作答案，把這份標註資料 (labelled dataset) 餵給監督式機器學習分類器 (supervised ML classifier)：模型學的不是某個固定規則，而是「對準動脈的條紋長什麼樣」——通常看脈動波形是否規律、條紋對比是否夠強、上下管壁邊界是否清楚。訓練完後，給它一條新的 M-mode 就能輸出「這條對到動脈」的機率，把 32 條過濾成最可信的那一條再去量直徑。為什麼這一步不能省？如果只做 32-channel 陣列、沒有 ML 自動篩選，等於每秒生出 32 條波形要使用者自己盯著挑，這跟傳統超音波仍要技師看畫面沒兩樣，穿戴貼片「不需要操作員 (operator-independent)」的承諾就破功了。ML 自動篩選把「物理冗餘」轉成「真正自動」，是這套設計的關鍵接合處。
   但同一個模型要能跨人用又是另一道關。每個人手腕的脂肪厚度、皮膚反射、動脈搏動幅度都不一樣，標註資料裡的條紋亮度、上下管壁雜訊樣式跟新使用者不會完全一樣。直接把模型套到新人身上，常會把對準的訊號判成沒對到、或反過來。作者再加一層領域適應 (domain adaptation)：這是一種專門解「訓練資料跟實際使用對不上」的技術，把標註資料學到的「對準動脈的特徵」搬到沒有標註的新使用者資料上，盡可能讓兩邊的特徵分布對齊。少了 domain adaptation，貼片就會一直挑錯 channel、量到根本不是動脈的訊號，吐出來的血壓波形完全失真——對 wearable 這種要服務無數使用者的場景，等於系統根本沒辦法部署到新人身上。
4. 工具與材料: 
   - **M-mode**: 把一條掃描線的回波隨時間排成 1D-time 影像；條紋寬度反映動脈直徑波形。
   - **32-element 陣列同時生成 32 條 M-mode**: 硬體冗餘設計，確保至少有一條對準動脈 (Lin et al. 2024)。
   - **Supervised ML classifier (監督式機器學習分類器)**: 用標註資料訓練，自動判斷哪一條 M-mode 對準動脈。
   - **Labelled dataset (標註資料集)**: 由人標好「對準/沒對到」的 M-mode 影像作為訓練答案。
   - **Domain adaptation (領域適應)**: 把標註資料學到的特徵搬到沒有標註的新使用者資料上，跨個體推廣。
   - **Operator-independent 量測**: 整個量測流程不需要操作員介入，全自動完成。
5. 與此篇文章的關係: 
   在《Wearable ultrasound technology》這篇文章中，作者為了讓穿戴血壓貼片真的能 operator-independent 工作，採用了 32-channel M-mode 陣列加監督式 ML 分類器再加 domain adaptation 的三段組合。這套方案解決了單一掃描線會因使用者活動偏離動脈、且 ML 在跨人時失準的瓶頸；它吃進 32 條 M-mode 訊號，自動產出「對準動脈」那一條的直徑波形，交給下游的「血管直徑→血壓」量化模型 (3-I 節) 換算成連續血壓。
